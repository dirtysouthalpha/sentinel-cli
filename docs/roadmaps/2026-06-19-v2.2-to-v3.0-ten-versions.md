# Sentinel CLI — v2.2 → v3.0 Roadmap (10 versions)

**Date:** 2026-06-19
**Premise:** The engine spine is at parity with Claude Code / opencode / Cursor. The competitive gaps are almost all **"built but not wired"** features. This roadmap closes each gap in priority order, one upgrade per version, each independently shippable.

Each version follows the pure-helper-first TDD discipline, ships with tests, and lands on `main` with a tag + release.

---

## v2.2 — Multi-model routing (the claimed moat, finally real)

**Gap:** The router engine (`src/ai/router.ts`) supports tiered routing by `taskKind`/`requiresVision`/`contextTokens`, but the call sites only ever pass `requiresTools`. `small_model` is configured but never consulted. `resolveRole` (`plan`/`smol`/`commit`) is dead code.

**Deliverables:**
- Populate `taskKind` in `RoutedProvider.resolveChain` — classify each turn (simple-read → smol, complex-reasoning → default, vision → vision-capable, commit-message → commit role).
- Route simple reads + chat to `small_model` (config) — ~3-5× cost cut for the bulk of turns.
- Wire `resolveRole("plan"|"smol"|"commit")` at the call sites (GSD plan phase → plan role, commit message → commit role).
- Tests: a pure `classifyTurn(messages, hasTools)` helper + router tests asserting the cheap model is picked for simple reads.

**Done when:** a session with 80% simple reads costs <40% of an all-default-model session, verified by the usage tracker.

---

## v2.3 — First-class GitHub PR tool

**Gap:** PR creation is a markdown prompt that shells `gh` blindly via bash — no auth check, no retry, no labels/assignees, no conflict detection.

**Deliverables:**
- New `pr` tool (`src/tools/pr.ts`): `gh` presence + auth check, end-to-end `create` (branch → push → `gh pr create` → return URL), `list`, `view`, `add-label`, `assign`, `merge`.
- Conflict detection: surface `<<<<<<<` markers from a failed merge to the model as structured hunks.
- Wire the existing `worktree.ts` + `agent-team.ts` (currently unused) into an `/team` orchestrator command that fans tasks across worktrees and merges back.
- Tests: pure argv builders + a fake `gh` runner (same pattern as `worktree.ts`).

**Done when:** `sentinel run "add a README and open a PR"` produces a real PR URL end-to-end without bash.

---

## v2.4 — Persistent memory (the agent remembers)

**Gap:** Nothing the agent concludes in one session survives to the next. `brain-recall.ts` is read-only with no store; `repo-index.ts` (TF-IDF) is built but never queried by the loop.

**Deliverables:**
- `brain_store` tool — persists facts/decisions/preferences to `.sentinel/memory.json` (structured, append-only, capped).
- Wire `repo-index.ts` into the system prompt as retrieval — top-K relevant code snippets by TF-IDF cosine rank, injected at turn start.
- Recall+store round-trip: the agent stores a fact, and a later session recalls it.
- Tests: pure memory-store CRUD + repo-index query against a fixture tree.

**Done when:** store a project decision in session A, recall it in a fresh session B with no context.

---

## v2.5 — Diff-at-approval gate (trust)

**Gap:** In `auto`/`gated` mode the user is asked "allow this edit?" but shown **no diff** — they approve blind. `renderWriteDiff` and the permission engine both exist; they're not joined.

**Deliverables:**
- When `permissions` returns `ask` for a file mutation, render the actual diff (LCS) in the prompt so the user sees what will change.
- Batch-approve: "show all N edits this turn, approve the batch."
- Per-hunk accept/reject in the GUI (extend `gui/src/render-diff.ts`).
- Tests: pure `formatApprovalPrompt(oldContent, newContent)` helper.

**Done when:** an edit in gated mode shows a real diff and the user can approve/reject without reading raw file paths.

---

## v2.6 — Real TDD mode (red-green-refactor)

**Gap:** GSD orders `implement → test` (code first). The `fix` phase regex-matches failure signals in prose, not actual test-runner exit codes.

**Deliverables:**
- Reorder GSD to `plan → test(red) → implement → test(green) → review`.
- Feed actual test-runner exit codes / output into the `fix` gate (not prose regex).
- New `/tdd <task>` command + `--tdd` flag on `sentinel run`/`sentinel loop`.
- Tests: pure `parseTestRunnerOutput(stdout, exitCode)` → structured pass/fail/failures.

**Done when:** `sentinel run --tdd "add input validation"` writes a failing test first, watches it fail, implements, watches it pass.

---

## v2.7 — Proactive budgets + live pricing

**Gap:** `overBudget()` is reactive; pricing is a static table that silently falls back to wrong defaults for new models.

**Deliverables:**
- Warn at configurable % thresholds (50/80/100%) of `budgetUSD`, not just after crossing.
- Live pricing fetch (TTL-cached) for unknown models, falling back to the static table.
- Surface interactive-session budgets (abort a turn that would blow the budget), not just autopilot.
- Tests: pure `budgetThresholds(spent, budget)` + pricing-cache TTL logic.

**Done when:** a session approaching its budget prints a warning at 80% and aborts cleanly at 100%.

---

## v2.8 — Cross-turn cache + multi-provider prompt caching

**Gap:** Tool results aren't memoized across turns (re-reading a file re-pays). Only Anthropic gets `cache_control` breakpoints.

**Deliverables:**
- Content-hash cache for `file:read` / `search` / `web` across turns (invalidate on file mtime).
- Add `cache_control` for OpenAI/Gemini/ZAI providers where their APIs support it.
- Tests: pure cache-key + invalidation logic.

**Done when:** reading the same file 5 times in a session costs 1 read; prompt-caching headers present on all providers that support them.

---

## v2.9 — Session branching + fork-from-turn

**Gap:** Resume exists; branching (explore alternatives without losing the trunk) doesn't.

**Deliverables:**
- `forkSession(id, turnIndex)` in `session-manager.ts` — copy messages up to N, diverge from there.
- `/fork` command in TUI/GUI; "branch from here" affordance on any turn.
- Tests: pure `forkMessages(messages, upToIndex)`.

**Done when:** fork at turn 5, take a different path, original session 5+ is preserved.

---

## v3.0 — Real plugin/tool extensibility + registry

**Gap:** The marketplace handles only `skill` and `mcp`; no third-party tools/themes/hooks. No default registry URL.

**Deliverables:**
- Extend `MarketplaceEntry.type` to `skill | mcp | tool | theme | hook`.
- On-permit hook semantics (allow/deny/rewrite) in `hooks.ts`.
- Ship a default registry URL; install/verify flow.
- Tests: pure entry-validation + hook-rule matching.

**Done when:** `sentinel install <tool-plugin>` from the registry registers a real tool.

---

## Goal-prompt compatibility note

These 10 versions are designed to be delivered by a single long-running `sentinel loop` (the daemon). Each version is scoped to ~1-3 files of net change following the pure-helper-first pattern, so the loop's PLAN step can decompose them cleanly. The goal prompt (below) instructs the loop to deliver them in order, one version at a time, tagging each release.

# Sentinel CLI v2.0 — Codebase Intelligence Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the four gaps that separate Sentinel CLI v1.2 (45 commits, 613 tests, security+TUI/GUI+autonomy shipped) from the 2026 leaders (opencode, Claude Code, Aider): **LSP-grade structural awareness, real model-driven compaction wired into the loop, true parallel multi-agent teams with git-worktree isolation, and a vision/multimodal surface that's first-class** — plus the production-readiness to call it v2.0 (release automation, the GitHub merge, CI).

Each ships as a tested pure module first, then a thin binding in `app.ts`/`main.ts`/`cli.ts`. No new stack: same TS/ESM/Blessed/vanilla-GUI, same `provider/model` routing, same keyring + sandbox + permission engine. Version bumps to `2.0.0`.

**Tech Stack:** TypeScript ESM, Node ≥20, Blessed (TUI), vanilla TS + WS (GUI), vitest, tsup, bubblewrap (sandbox), MCP SDK. New dep: `vscode-languageserver` (LSP client) — optional/dynamic, same pattern as puppeteer.

**Baseline gates (must be green before starting):** `npm run lint` clean, `npm test` 613 passing, `npm run build` succeeds, GUI Vite build succeeds.
